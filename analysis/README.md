# `./analysis/` Directory

One folder per finished (or in-progress) map project. Each folder is self-contained: it holds the rendering script(s), a README describing the map's intent and data sources, and any map-specific notes.

## Conventions

1. Folder names are short and descriptive: `edmonton-neighborhoods/`, `alberta-rivers/`, `canada-population-density/`.
2. Every map folder contains at minimum one `.R` or `.py` rendering script that runs top-to-bottom without manual steps.
3. Finished map images export to `data-public/derived/maps/` — not stored inside `analysis/`.
4. Document projection rationale and design decisions in the folder's `README.md`.
